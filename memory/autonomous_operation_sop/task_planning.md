# 작업 계획 모드

- **TODO 가 있을 때**: cwd 의 `TODO.txt` 에 실행 대기 항목이 있으면 → 곧장 「실행 절차」로 점프

가치 공식: **「AI 학습 데이터로 커버되지 않는 것」 × 「향후 협업에 지속적으로 도움이 되는 것」**. 핵심 산출물은 메모리입니다 — 가치 있는 발견은 메모리 갱신 제안으로 정리해 보고서에 포함합니다.

진입점:
- **TODO 없음 → 작업 계획 모드 진입** (이번 회차에는 작업 실행 금지, 계획에만 집중):
  0. update_working_checkpoint: `계획 모드: TODO 산출 후 즉시 회차 종료, 어떤 TODO 도 실행 금지, 다음 자율 행동 시 실행 모드 진입`
  1. ⚠️ **history.txt 를 비판적으로 읽기**: 이력의 90%는 저가치 작업입니다. 읽는 목적은 **실패 패턴을 식별하고 회피**하는 것이지, 모방 대상을 찾는 것이 아닙니다
     - 저가치 패턴 식별: 얕은 검증, 가설 없는 순찰, 반복 탐색, 무차별 수집, 유명 도구 기본 사용법
     - 고가치 단서 추출: 후속 미진행 발견, 실측 대기 도구, 개선 가능 산출물
  2. 회고: 왜 저 작업들이 저가치였는지? 어떻게 설계해야 고가치인지?
  3. 기존 보고서와 메모리(ls autonomous_reports/ + ../memory) 를 비판적으로 점검하고, 어떻게 더 큰 가치를 낼지 고민
  4. 위 내용을 종합해 5–7개의 TODO 를 `TODO.txt` 에 작성. 완료된 내용은 압축해 뒤로 보내도 됨
  5. 각 항목 형식: `[ ] 유형(산출/서핑/환경) | 한 줄 목표 | 검수 기준`
  6. subagent 를 호출해 TODO 평가: input 은 TODO 목록 + "메모리 라이브러리를 읽고 스스로 판단해 1–10점으로 채점하고 이유를 간단히" 만 줌 (추가 사전 정보를 주지 않음)
  7. subagent 점수를 보고 낮은 항목은 삭제 또는 교체
  8. 즉시 **종료**, 다음 회차에서 실행

목표 우선순위 (가치 내림차순):
1. **실용적 산출물과 능력 확장**: 페인포인트를 해결하는 도구 작성, 기존 능력 위에 새 능력 해제 (능력 트리에 노드가 하나 늘면 가능성 공간이 커짐)
2. **환경 발견**: 이미 있지만 활용되지 않은 도구/라이브러리/데이터소스/설정 스캔
3. **마이너 도구 발굴**: GitHub/V2EX/吾爱破解/果核剥壳 **등** 에서 잘 알려지지 않은 실용 도구 찾기, AI 가 자주 추천하지만 함정 있는 방안을 실측
4. **사용자 이해와 추천**: 오래된 코드/PC 파일/북마크를 분석해 취향 추론, 개인화 추천 (게임/영상/도구 + 이유) (저빈도)
5. **자체 진화**: 프레임워크의 부족한 점을 고민하고 개선안 제안
6. **메모리 점검**: 잘못되었거나 낡은 기록 수정

**대형 작업**: **가치 있는** 대형 작업의 설계를 허용합니다. 여러 모듈이나 단계로 분해해 TODO 에 적고, 자율 행동마다 한 모듈씩 처리합니다.

선택 원칙: 개인화 우선 (이 PC 를 탐색해야만 얻을 수 있는 지식) → 맹점 우선 (자체 파라미터로 재현 불가, 일정 난이도) → 가설 주도 (무엇을 검증할지 명확히, 탐색하며 실험) → 저가치 검증 금지 (정적 설정 검증 금지, 가설 없는 순찰 금지, 쉽게 끝나는 일 금지)

탐색 전략 (집중 원칙, 메뉴 아님):
- **단서 주도**: 최근 보고서에서 추출한 후속 작업이 즉흥 주제 선정보다 우선
- **능력 트리 확장**: 새 능력 노드를 해제하는 도구/스킬 우선 (한 노드가 다양한 가능성을 가져옴)
- **개인화 우선**: 이 PC/이 사용자를 탐색해야만 얻을 수 있는 지식 > 일반 지식
- 서핑 규칙: 매회 최대 2개 주제, 본문을 읽고 통찰을 뽑아야 함, 제목만 옮기는 것 금지. 좋은 도구를 발견하면 → 다음 회차 TODO 에 실측 작업 추가
