## 언제 쓰는가
정기적으로 자동 실행되어야 하는 작업 (일일 리포트, 주간 점검, 1회성 알림 등).

## 작동 원리 (Why)
`scheduler.py`(reflect/)가 60초마다 `sche_tasks/*.json`을 폴링, 모든 조건 충족 시에만 트리거 — enabled + 현재 시간 ≥ schedule + 쿨다운 경과.

## 디렉터리

- `../sche_tasks/` — 작업 정의 JSON
- `../sche_tasks/done/` — 실행 보고서

## 작업 JSON 형식

```json
{
  "schedule": "08:00",
  "repeat": "daily",
  "enabled": true,
  "prompt": "...",
  "max_delay_hours": 6
}
```

**repeat 가능 값**: `daily` | `weekday` | `weekly` | `monthly` | `once` | `every_Nh` (N시간마다) | `every_Nd` (N일마다)

**max_delay_hours** (기본 6): schedule 보다 N시간 지나면 트리거 안 함. 부팅 늦어 옛 작업 실행 방지.

## 트리거 흐름

1. scheduler가 60초마다 `sche_tasks/*.json` 폴링
2. 조건 충족 시 prompt 조립 + 보고서 경로 `../sche_tasks/done/YYYY-MM-DD_작업명.md` 주입
3. **작업 수신 후 첫 단계**: `update_working_checkpoint`로 보고서 경로 기록 (긴 작업 도중 잊지 않도록)
4. 실행 완료 후 위 경로에 보고서 작성 (scheduler가 이 파일로 오늘 실행 여부 판정)

## 로그와 모니터링

- `sche_tasks/scheduler.log` — 트리거/스킵/에러 자동 로그
- `scheduler.health_check()` — HEALTHY/OVERDUE/DISABLED/NEVER_RUN/ERROR 상태 목록
- JSON 파싱 오류, schedule 형식 오류, 알 수 없는 repeat 모두 로그 기록

## 자주 빠지는 함정

<Warning>
- **once 유형**: 1회 실행 후 100년 쿨다운 (실효는 영구 스킵)
- **sche_tasks 위치**: `../`, 즉 코드 root 아래 (memory/ 가 아님)
- **보고서 경로는 자동 생성**: 작업 파일은 "무엇을 할지"만, 보고서 경로는 scheduler가 prompt에 주입
- **첫 단계에서 checkpoint에 보고서 경로 기록 안 하면** 긴 작업 도중 경로 잊고 다른 곳에 보고서 작성하는 사고 발생
</Warning>
