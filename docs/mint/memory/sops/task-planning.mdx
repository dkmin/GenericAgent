## 언제 쓰는가
자율 행동 진입 시 cwd의 `TODO.txt`에 실행 대기 항목이 없을 때. 이번 회차는 **계획에만 집중**, 작업 실행 금지.

## 작동 원리 (Why)
이력의 90%는 저가치 작업입니다. 그냥 따라하면 시간만 낭비. 계획 모드의 핵심은 **저가치 패턴을 식별하고 회피**, 고가치 단서만 다음 회차로 넘기는 것.

## 진입 흐름

1. checkpoint: `계획 모드: TODO 산출 후 즉시 회차 종료, 어떤 TODO도 실행 금지`
2. **history.txt 비판적 읽기**:
   - 저가치 패턴 식별: 얕은 검증, 가설 없는 순찰, 반복 탐색, 무차별 수집, 유명 도구 기본 사용법
   - 고가치 단서 추출: 후속 미진행 발견, 실측 대기 도구, 개선 가능 산출물
3. 회고: 왜 저가치였는지? 어떻게 고가치인지?
4. `ls autonomous_reports/ + ../memory` 로 기존 점검
5. 5-7개 TODO를 `TODO.txt` 작성. 형식: `[ ] 유형(산출/서핑/환경) | 한 줄 목표 | 검수 기준`
6. **subagent로 TODO 평가**: input은 TODO 목록 + "메모리 라이브러리 읽고 1-10점, 이유 간단히"만 (사전 정보 X)
7. 점수 낮은 항목 삭제 또는 교체
8. 즉시 **종료**

## 목표 우선순위 (가치 내림차순)

1. **실용적 산출물·능력 확장**: 페인포인트 해결 도구, 능력 트리 노드 +1
2. **환경 발견**: 이미 있지만 미활용 도구/라이브러리/데이터
3. **마이너 도구 발굴**: GitHub/V2EX/吾爱破解 등 잘 알려지지 않은 실용 도구
4. **사용자 이해와 추천**: 코드/PC 파일/북마크 분석으로 취향 추론
5. **자체 진화**: 프레임워크 부족점 개선안
6. **메모리 점검**: 잘못/낡은 기록 수정

## 선택 원칙

- **단서 주도** > 즉흥 주제 (최근 보고서 후속 작업 우선)
- **개인화 우선** (이 PC/이 사용자 탐색해야만 얻는 지식 > 일반 지식)
- **가설 주도** (무엇을 검증할지 명확히)
- **저가치 검증 금지** (정적 설정 검증, 가설 없는 순찰, 쉽게 끝나는 일)

## 자주 빠지는 함정

<Warning>
**금지 영역**:
- Hacker News · 뉴스 헤드라인 훑기
- 무차별 헤드라인 수집/목적 없는 뉴스 새로고침
- 유명 도구 기본 사용법 탐구
- 현재 프레임워크보다 약한 agent 조사
- 다른 web 자동화/computer use 프레임워크 조사
- 자체 코드베이스 읽기
</Warning>

<Tip>
서핑 규칙: 매회 최대 2개 주제, 본문 읽고 통찰 추출, 제목만 옮기기 금지. 좋은 도구 발견 시 → 다음 회차 TODO에 **실측 작업** 추가.
</Tip>
