## 언제 쓰는가
API key, password 등을 코드에서 사용해야 하는데 **로그/터미널에 평문 노출되면 안 될 때**.

## 작동 원리 (Why)
Agent의 라운드 출력은 모두 로그로 남습니다. API key를 변수로 그대로 두면 어딘가에서 print되거나 stack trace에 노출되기 쉽습니다. `keychain`은:

- 디스크에 XOR 마스킹 저장 (`~/ga_keychain.enc`)
- repr/str을 가로채 **자동으로 마스킹된 미리보기**만 표시
- 실제 값은 `.use()` 호출로만 추출

## API

| 메서드 | 설명 |
|---|---|
| `keys.set(name, value)` | 메모리에 등록 + 디스크 저장 |
| `keys.set(name, file=path)` | 파일에서 읽어 저장 |
| `keys.NAME` | `SecretStr` 반환 (repr는 마스킹) |
| `keys.NAME.use()` | 실제 raw 값 (print 금지) |
| `keys.ls()` | 등록된 이름 목록 |

## 사용 예시

<CodeGroup>
```python 등록
import sys
sys.path.append('../memory')
from keychain import keys

keys.set("openai", "sk-...")
keys.set("anthropic", file="/path/to/key.txt")
```

```python 사용
from keychain import keys

# 안전: repr가 마스킹되어 print/log에 노출 안 됨
print(keys.openai)
# → SecretStr(openai=sk-···xxx) # .use() to get raw, do not print raw value

# 실제 사용
import openai
openai.api_key = keys.openai.use()  # 절대 print 하지 말 것
```
</CodeGroup>

## SecretStr 마스킹 규칙

| 길이 | 미리보기 |
|---|---|
| ≤ 4 | `***` |
| ≤ 16 | `abc···xyz` |
| ≤ 40 | `abcdef···xyz123 len=N` |
| > 40 | `abcdefghij···xyz123 len=N` |

## 자주 빠지는 함정

<Warning>
- **`.use()` 결과를 print하지 말 것** — 마스킹의 의미가 사라짐
- 디스크 파일이 손상되면 자동으로 `.enc.bak`으로 백업되고 빈 keychain으로 시작 — `keys.set` 다시 호출 필요
- XOR 마스크는 **사용자명 기반** — 동일 OS 계정에서만 복호화 가능 (다른 머신/계정에 복사하면 못 읽음)
</Warning>
