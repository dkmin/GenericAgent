<Note>유휴 자율 행동은 두 가지 컴포넌트의 합입니다 — `launch.pyw`의 `idle_monitor` 스레드(트리거)와 `memory/autonomous_operation_sop.md`(행동 규약). 사용자는 사이드바에서 활성화 여부만 토글합니다.</Note>

## 트리거 동작

`launch.pyw` 65–79번째 줄의 `idle_monitor` 스레드가 5초 주기로 페이지의 `#last-reply-time`을 검사합니다.

```python
if now - last_reply > 1800:
    print('[Idle Monitor] Detected idle state, injecting task...')
    inject("[AUTO]🤖 用户已经离开超过30分钟，作为自主智能体，请阅读自动化sop，执行自动任务。")
```

즉, **마지막 응답 후 1800초(30분) 경과 시 한 번** 자동 메시지가 chat input에 주입되며, 다음 트리거까지 최소 120초 쿨다운이 적용됩니다.

## 사이드바 토글

<Steps>
  <Step title="자율 행동 허용">
    사이드바의 **▶️ 자율 행동 허용**을 누르면 `st.session_state.autonomous_enabled = True`로 바뀌고, 동시에 `last_reply_time`을 1800초 전으로 셋해 즉시 트리거되도록 합니다.
  </Step>
  <Step title="유휴 시간 모니터링">
    사이드바 상단의 "유휴 시간: NNN초" 캡션으로 현재 카운트를 확인할 수 있습니다.
  </Step>
  <Step title="중지">
    **⏸️ 자율 행동 중지**를 누르면 토글이 꺼집니다. (스레드는 계속 살아있지만 SOP 규약상 동의 없이는 행동하지 않습니다.)
  </Step>
</Steps>

## 진입점: autonomous_operation_sop.md

자동 inject된 메시지는 에이전트에게 `memory/autonomous_operation_sop.md`를 읽으라고 지시합니다. 이 SOP의 핵심은 다음과 같습니다.

- **시작**: `update_working_checkpoint`로 마무리 시 SOP 재독을 명시
- **이력 확인**: `from autonomous_operation_sop.helper import *; print(get_history(40)); print(get_todo())`로 중복 회피
- **작업 선택 원칙**: 미완료 TODO 1건만, 같은 하위 작업 연속 선택 금지
- **가치 공식**: 「AI 학습 데이터로 커버되지 않는 것」 × 「향후 협업에 지속적으로 도움이 되는 것」
- **마무리 4단계**: SOP 재독 → 보고서 작성 → `complete_task(...)` → `set_todo()`로 완료 표시
- **권한 경계**: 비밀키, 핵심 코드, 비가역 작업은 절대 금지. global memory/SOP 수정·외부 API 호출은 보고서로 검토 요청

<Warning>자율 행동 중에는 `./autonomous_reports/RXX_*.md` 보고서가 누적됩니다. 사용자가 돌아왔을 때 검토하여 승인/수정/거부하는 것이 워크플로우의 일부입니다.</Warning>

## 끄고 싶을 때

- 일시적으로: 사이드바에서 **자율 행동 중지**
- 영구적으로: `launch.pyw`의 `monitor_thread = threading.Thread(target=idle_monitor, daemon=True); monitor_thread.start()` 두 줄을 주석 처리

<Tip>30분 임계값이 너무 짧다면 `launch.pyw`에서 `if now - last_reply > 1800:` 의 1800을 원하는 초 단위 값으로 수정하세요.</Tip>
