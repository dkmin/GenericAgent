> 🤖 **Self-Bootstrap Proof** — Everything in this repository, from installing Git and running `git init` to every commit message, was completed autonomously by GenericAgent. The author never opened a terminal once.
>
> — [README.md](https://github.com/dkmin/GenericAgent/blob/main/README.md#L22)

## 무엇을 의미하는가

이건 마케팅 문구가 아닙니다. **레포 전체가 데모(데몬스트레이션)이자 증거**입니다.

평범한 자가진화 에이전트라면 — "내일 알람을 설정해줘" 같은 일을 한 번 하고, 그 경로를 Skill로 저장합니다. GenericAgent는 한 단계 더 나갑니다. **자기 자신의 개발 환경을 자기가 부트스트랩**합니다.

```
[빈 폴더] →
  Git을 설치한다 (code_run으로) →
  git init →
  파일들을 작성한다 (file_write로) →
  pip install 의존성 →
  자기 자신을 launch.pyw로 실행해 검증 →
  git add . && git commit →
  GitHub 레포 생성 (web_execute_js로 GitHub UI 조작) →
  git push →
  README.md를 자기가 쓴다
```

작가는 "이런 에이전트를 만들고 싶다"는 자연어 지시만 줬습니다. **나머지는 9개 원자 도구의 합성**이 처리했습니다.

## 왜 가능한가 — 3개 기둥

1. **원자 도구 9개로 충분** — `code_run` 하나면 `git`, `pip`, `bash` 가 모두 종속됩니다. task-specific 도구를 미리 만들 필요가 없습니다.
2. **계층형 메모리 (L0–L4)** — 한 번 배운 git 워크플로우는 L3 SOP로 저장되어 다음 작업에서 한 줄로 재호출됩니다.
3. **조립 가능한 시스템 프롬프트** — Meta Rules (L0) + Insight Index (L1) + 현재 SOP만 매 턴 주입 → 30K 컨텍스트로도 충분.

## 의미

- **Skill을 미리 만들 필요가 없습니다.** GenericAgent는 처음 본 작업도 한다. 그리고 그 경로를 결정화해 다음번엔 한 줄로 호출합니다.
- **3K 줄짜리 시드코드가 자기 자신을 키웁니다.** 사용 시간이 길어질수록 당신만의 Skill 트리가 자랍니다 — 누구도 가지지 못한 것.
- **터미널을 모르는 사용자도 코드 베이스를 가질 수 있습니다.** Self-Bootstrap은 곧 진입 장벽 0을 의미합니다.
