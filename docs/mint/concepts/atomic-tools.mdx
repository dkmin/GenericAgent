## 시드 도구 카탈로그

`assets/tools_schema.json`에 정의된 도구는 단 9개입니다. 모두 **atomic** — 합성하지 않은 원자 단위입니다.

```json
[
  {"name": "code_run"},
  {"name": "file_read"},
  {"name": "file_patch"},
  {"name": "file_write"},
  {"name": "web_scan"},
  {"name": "web_execute_js"},
  {"name": "update_working_checkpoint"},
  {"name": "ask_user"},
  {"name": "start_long_term_update"}
]
```

## "더 만들지 마라" 원칙

새 도메인이 와도 **도구를 추가하지 않습니다**. 대신:

<Tabs>
  <Tab title="잘못된 접근">
    ```python
    # ❌ "WeChat 메시지 읽기" 도구를 새로 만든다
    @tool
    def read_wechat_messages(...): ...

    # ❌ "주식 스크리닝" 도구를 새로 만든다
    @tool
    def screen_stocks(...): ...
    ```
    - 도구 카탈로그가 비대해지고 컨텍스트가 부풀어오릅니다.
    - 모든 사용자에게 같은 도구가 강제됩니다.
  </Tab>
  <Tab title="올바른 접근 (GenericAgent)">
    ```text
    memory/
    ├─ wechat_read_sop.md       ← 에이전트가 작성한 SOP
    ├─ stock_screen_sop.md      ← 에이전트가 작성한 SOP
    └─ ljqCtrl.py               ← 재사용 가능한 helper 모듈
    ```
    - 도구는 9개 그대로. 도메인은 SOP + helper 파이썬 모듈로 확장합니다.
    - SOP는 `code_run`이 호출할 helper만 안내하면 됩니다.
  </Tab>
</Tabs>

실제 helper 예시는 `memory/`에 들어있습니다:
- `adb_ui.py` — Android 디바이스 UI 제어 helper
- `ljqCtrl.py` — 키보드/마우스 정밀 제어
- `procmem_scanner.py` — 프로세스 메모리 스캔
- `ocr_utils.py`, `ui_detect.py` — 화면 비전

## 디스패치 흐름

`agent_loop.py`의 `BaseHandler.dispatch`가 도구를 받아 `do_<tool_name>` 메서드로 라우팅합니다.

```python
def dispatch(self, tool_name, args, response, index=0):
    method_name = f"do_{tool_name}"
    if hasattr(self, method_name):
        ret = yield from try_call_generator(getattr(self, method_name), args, response)
        return ret
    elif tool_name == 'bad_json':
        return StepOutcome(None, next_prompt=args.get('msg', 'bad_json'), should_exit=False)
```

새 도구를 추가하려면 `do_<name>` 메서드와 스키마 항목만 추가하면 되지만 — **굳이 추가하지 마세요.** 9개로 충분합니다.
