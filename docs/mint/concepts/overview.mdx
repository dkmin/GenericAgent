## 3K줄의 시드 코드

GenericAgent는 핵심 코드가 약 **3,000줄**에 불과합니다. 이 안에 LLM이 로컬 컴퓨터의 브라우저, 터미널, 파일시스템, 키보드/마우스, 화면 비전, 모바일(ADB)까지 제어할 수 있는 모든 권한이 들어있습니다.

```text
agent_loop.py     ~123줄   에이전트 루프 (turn 관리 · tool dispatch)
ga.py             ~560줄   핸들러 · 메모리 인젝션 · 시스템 프롬프트 조립
agentmain.py      ~268줄   세션 진입점
llmcore.py        ~982줄   LLM 어댑터 (Claude/Gemini/Kimi/MiniMax)
```

비결은 단순합니다. **도구 개수를 늘리는 대신, 메모리에 SOP를 누적합니다.**

## 왜 3K줄인가

다른 에이전트 프레임워크는 도구를 늘리는 방향으로 진화했습니다. "Gmail 보내기 도구", "WeChat 메시지 도구", "주식 스크리닝 도구"… 결과적으로 코드는 늘어나지만 새 도메인을 만나면 결국 새 도구를 또 만들어야 합니다.

GenericAgent는 반대로 갑니다. **도구는 9개로 고정**하고, 새 도메인을 만나면 에이전트가 스스로 SOP 마크다운을 작성해 `memory/`에 저장합니다. 이 SOP는 다음번 동일 트리거에서 자동 호출됩니다. 코드 베이스는 늘지 않고, 능력만 늡니다.
